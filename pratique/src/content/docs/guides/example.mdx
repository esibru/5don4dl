## Écrire un nouveau laboratoire

Pour écrire un nouveau laboratoire, il suffit d'ajouter un fichier mdx dans le répertoire `src/content/docs/laboratoires`.
Une première page blanche est disponible pour Redis (`redis.mdx`).

`mdx`est une version enrichie de `md` dans lequel on peut ajouter des "composants". Ce guide est fait pour comprendre leur utilisation dans les grandes lignes.

## astro.config.mjs

En principe, il n'est pas nécessaire de modifier ce fichier. Tout ce qui se trouve dans le répertoire `src/content/doc/laboratoires` est automatiquement ajouté dans la table des matières (un titre par fichier type markdown ou mdx).

Une raison de modifier ce fichier est d'ajouter un répertoire cousin.

## `index.mdx`

Page d'accueil du site. En principe, il ne faut pas le modifier.

### Étapes

<Steps>

1. Initialiser le dépôt

   ```sh showLineNumbers=false
   git init
   ```

1. Écouter tous les fichiers
  
   ```bash showLineNumbers=false
   git add .
   ```
1. Faire un commit
  
   ```shell title="commit" showLineNumbers=false
   git commit -m "initial commit"
   ```
1. Et bien sûr, faire un push
  
   ```shell showLineNumbers=false
   git push
   ```
</Steps>
